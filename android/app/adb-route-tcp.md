Чтобы перенаправить TCP-порт на конкретный эмулятор Android с помощью `adb`, выполните следующие шаги:

---

### 🔧 Перенаправление порта с использованием `adb forward`

Команда `adb forward` позволяет направить трафик с порта хост-машины на порт эмулятора или устройства:

```bash
adb -s <emulator_serial> forward tcp:<host_port> tcp:<device_port>
```



* `<emulator_serial>` — идентификатор эмулятора, например, `emulator-5554`.
* `<host_port>` — порт на вашей хост-машине.
* `<device_port>` — порт на эмуляторе, на который будет перенаправлен трафик.([hjortsberg.org][1])

**Пример:**

Если вы хотите перенаправить порт 8080 хост-машины на порт 3000 эмулятора:

```bash
adb -s emulator-5554 forward tcp:8080 tcp:3000
```



После этого запросы, отправленные на `localhost:8080` вашей хост-машины, будут перенаправлены на порт 3000 эмулятора.

---

### 🔁 Обратное перенаправление с использованием `adb reverse`

Если необходимо, чтобы эмулятор обращался к сервису на хост-машине, используйте команду `adb reverse`:

```bash
adb -s <emulator_serial> reverse tcp:<device_port> tcp:<host_port>
```



**Пример:**

Чтобы эмулятор мог обращаться к сервису на порту 8080 хост-машины через свой порт 3000:

```bash
adb -s emulator-5554 reverse tcp:3000 tcp:8080
```



Теперь эмулятор, обращаясь к `localhost:3000`, будет фактически взаимодействовать с `localhost:8080` хост-машины.

---

### 📋 Просмотр текущих перенаправлений

Чтобы увидеть список всех текущих перенаправлений портов:

```bash
adb -s <emulator_serial> forward --list
```



Это отобразит все активные перенаправления для указанного эмулятора.

---

### 🧹 Удаление перенаправлений

Чтобы удалить конкретное перенаправление:

```bash
adb -s <emulator_serial> forward --remove tcp:<host_port>
```



Или для удаления всех перенаправлений:

```bash
adb -s <emulator_serial> forward --remove-all
```



---

### 📌 Дополнительные советы

* Убедитесь, что используете правильный идентификатор эмулятора. Вы можете получить список подключенных устройств и эмуляторов с помощью:

  ```bash
  adb devices
  ```



* Если вы запускаете несколько эмуляторов, обязательно указывайте `-s <emulator_serial>` для точного назначения перенаправлений.

* Команды `adb forward` и `adb reverse` полезны для тестирования приложений, особенно когда необходимо взаимодействие между эмулятором и сервисами на хост-машине.
